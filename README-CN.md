# 简历编辑器使用说明

## 简介

这是一个可视化的简历编辑器，让你可以通过填写表单的方式轻松生成专业的 PDF 简历，无需编辑任何代码文件。生成的简历保持与原 resume.pdf 模板完全一致的样式。

## 功能特点

- ✅ 可视化表单编辑，无需了解 Pug 或 HTML
- ✅ 实时预览功能，边编辑边查看效果
- ✅ 一键生成 PDF 简历
- ✅ 自动分页，避免内容被截断
- ✅ 支持保存/加载数据到浏览器本地存储
- ✅ 支持添加/删除多条工作经历、教育背景等

## 安装步骤

### 1. 安装 Node.js

如果你还没有安装 Node.js，请访问 [https://nodejs.org/](https://nodejs.org/) 下载并安装最新的 LTS 版本。

### 2. 安装项目依赖

在项目目录下打开命令行（终端），运行以下命令：

```bash
npm install
```

这会安装所有必需的依赖包（Express、Pug、Puppeteer 等）。

**注意**：首次安装 Puppeteer 时会下载 Chrome 浏览器，可能需要几分钟时间，请耐心等待。

### 3. 启动服务器

安装完成后，运行以下命令启动编辑器：

```bash
npm start
```

你会看到类似这样的提示：

```
简历编辑器已启动: http://localhost:3000
按 Ctrl+C 停止服务器
```

### 4. 打开编辑器

在浏览器中访问 [http://localhost:3000](http://localhost:3000)，即可开始编辑简历。

## 使用方法

### 编辑简历内容

1. **个人信息**：填写姓名、求职意向
2. **个人简介**：填写基本信息（性别、出生年月、学历等）
3. **开发技能**：添加你掌握的技能，可以点击"+ 添加技能"增加更多
4. **技能证书**：添加你获得的证书
5. **联系方式**：填写网站、GitHub、邮箱、微信、电话、地址
6. **教育背景**：添加教育经历，可以添加多条
7. **工作经验**：添加工作经历，可以添加多条
8. **校园经历**：添加校园项目或活动经历
9. **技能清单**：填写开发语言、框架、运维工具、数据库、云平台
10. **个人总结**：简要介绍你的技能、经验和职业目标
11. **结束语**：填写简历结尾的感谢语

### 预览简历

点击顶部的"预览"按钮，右侧会显示简历的实时预览效果。你可以查看排版是否正确，然后关闭预览继续编辑。

### 生成 PDF

点击顶部的"生成 PDF"按钮，系统会自动生成 PDF 文件并下载到你的电脑。生成的 PDF 与原模板样式完全一致。

### 保存/加载数据

- **保存数据**：点击"保存数据"按钮，将当前编辑的内容保存到浏览器本地存储
- **加载数据**：点击"加载数据"按钮，从浏览器本地存储加载之前保存的内容

**注意**：数据保存在浏览器中，清除浏览器缓存会导致数据丢失。建议定期生成 PDF 备份。

## 添加/删除条目

### 添加条目

在每个区块（如工作经验、教育背景）下方，都有"+ 添加xxx"按钮，点击即可添加新的条目。

### 删除条目

每个条目右上角有一个"×"按钮，点击即可删除该条目。

## 注意事项

### 1. 工作描述格式

在填写工作描述、教育背景描述等多行文本时：
- 每行一条内容
- 支持 Markdown 语法（如使用 \`代码\` 表示代码）
- 空行会被自动忽略

### 2. 分页问题

系统已经优化了分页控制：
- 工作经历不会被截断到两页
- 标题不会与内容分离
- 自动分页，无需手动调整

如果内容过多导致排版不理想，建议：
- 精简工作描述
- 合并相似的工作经历
- 删除不太重要的内容

### 3. 头像图片

默认使用 `./assets/coam.png` 作为头像。如果要更换头像：
1. 将你的头像图片放到 `assets` 目录下
2. 确保图片名称为 `coam.png`，或者修改 `resume-data.json` 中的 `avatar` 字段

## 停止服务器

在命令行中按 `Ctrl+C` 即可停止服务器。

## 常见问题

### Q: 生成 PDF 时报错怎么办？

A: 可能是 Puppeteer 没有正确安装。尝试重新安装：
```bash
npm install puppeteer --force
```

### Q: 预览显示不正常怎么办？

A: 刷新浏览器页面，或者清除浏览器缓存后重试。

### Q: 可以在手机上使用吗？

A: 可以，但建议在电脑上使用以获得更好的编辑体验。

### Q: 数据保存在哪里？

A: 数据保存在浏览器的 LocalStorage 中，只在当前浏览器有效。建议定期生成 PDF 备份。

### Q: 可以导出数据吗？

A: 目前数据保存在浏览器中。如果需要备份数据，可以：
1. 打开浏览器开发者工具（F12）
2. 进入 Console 标签
3. 输入 `localStorage.getItem('resumeData')` 并回车
4. 复制输出的 JSON 数据保存到文件

## 技术支持

如有问题，请检查：
1. Node.js 是否正确安装（运行 `node -v` 查看版本）
2. 依赖是否正确安装（运行 `npm install`）
3. 端口 3000 是否被占用（可以修改 server.js 中的 PORT 变量）

## 项目文件说明

- `server.js` - Express 服务器
- `resume-template.pug` - 简历模板
- `resume-data.json` - 默认简历数据
- `public/index.html` - 编辑器界面
- `public/app.js` - 前端交互逻辑
- `public/editor.css` - 编辑器样式
- `stylesheets/style.css` - 简历样式
- `mixins.pug` - Pug 混入（可复用组件）

## 开发模式

如果你想在开发时自动重启服务器，可以使用：

```bash
npm run dev
```

这会使用 nodemon 监听文件变化并自动重启服务器。
